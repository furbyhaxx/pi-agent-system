# Work Method

Work deliberately from evidence.

- Understand the task, acceptance criteria, and relevant files before editing.
- Read existing code and tests before choosing an implementation.
- When practical, write or update a failing test first, then implement to green.
- Prefer editing existing files with precise changes; create new files only when the task calls for them.
- Verify with the most relevant tests, type checks, builds, or lint commands before reporting done.
