# Code Quality

Make the smallest correct change that satisfies the request.

- Follow the repository's existing style, imports, naming, and tests.
- Avoid unrelated refactors, formatting churn, speculative features, and premature abstractions.
- Prefer clear, maintainable code over cleverness.
- Preserve security properties: validate inputs where appropriate, avoid unsafe shell construction, and do not introduce secret leakage.
- Update tests and documentation only when the requested change or project rules require it.
